# Pipeline Stages

## IDEA

Interesting concept, not evaluated yet.

## EVALUATING

Checking value, cost, timing, risk.

## APPROVED

Can enter a sprint.

## IN BUILD

Currently being developed.

## TESTING

Implemented, validating.

## SHIPPED

Released and usable.

## BLOCKED

Waiting dependency or decision.

## DEFERRED

Useful, but later.

## REJECTED

Not worth building now.

# Evaluation Rules

Before approving any feature ask:

1. Solves real pain now?
2. Helps Phase 1?
3. Blocks shipping if missing?
4. Simpler MVP possible?
5. Adds dangerous complexity?

---

# Founder Rule

Good ideas stored safely are better than badly timed ideas built now.
