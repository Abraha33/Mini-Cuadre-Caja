# ROADMAP.md

## Project

Mini Cuadre Caja

## Product Strategy

Ship a reliable operational core first.

Expand only after stability.

---

# PHASE 1 — Core Usable Product (Current Priority)

## Goal

A real business can use the app daily.

## Must Include

- Stable login
- Role loading
- Open shift
- Active shift persistence
- Basic daily operations
- Close shift
- Difference detection
- Final report export
- Basic offline continuity

## Success Definition

A cashier can operate start to finish in one day.

## Status

IN PROGRESS

---

# PHASE 2 — Reliability Layer

## Goal

Trust the system in real conditions.

## Includes

- Robust sync queue
- Retry engine
- Crash recovery
- Better logs
- Safer exports
- Core automated tests

## Status

NEXT

---

# Current Founder Rule

Do not leave Phase 1 unfinished to chase later phases.

---

# Decision Filter

Before starting anything new ask:

1. Does it help Phase 1 now?
2. Does it unblock shipping?
3. Can it wait?
4. Is there a smaller MVP version?
